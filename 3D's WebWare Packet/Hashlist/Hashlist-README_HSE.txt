HashedLists AKA Hashlists are web-stored lists but the data is stored as a hash into the link for quick and easy lists systems that can be changed on the fly while keeping your information private to just you. Our lists are not stored on a server and the only interaction with the server-as-a-node is to decrypt the hashlink the list-data is stored in, which if you choose to, you can unlock without the public server (visavis IttyBittySite Github).

	
	
The difference between Hashed-Stream-Elements (HSE) and standard Hashlists is the allowed characters. HSE does allow for more characters by just a few. HSE is less secure for this reason and that's why there's two different versions. The original HashLists doesn't allow for many characters and attempts to prevent links and scripts to prevent cross-device hashed-infections.


- "hashlist.html" is the standard version.

- "hashlist (old phone ver).html" is an older version just for historical purposes.

- "HashedStreamElements.html" is the 'advanced' version, please use with caution!
**HSE is designed to be able to copy and paste OBS web-based stream objects (normally links)**

 hashlist.html (Standard):
--------------------------------
 1) Type item in "Item" field
 2a) If you need something click "Need" but if you have something click "Have"
 2b) If you have something but need more: follow steps 2 & 3 for "Have" then repeat steps 2 & 3 for "Need"
 3a) If you Click "Need", you are done
 3b) If you Click "Have", you will need to next input how many of these you have then click "Add to Have"
 
 EXTRA) At anytime you can click on edit for holding items to adjust how many you have & almost all object names can be clicked on to rename (click change to keep all changes)

 HashedStreamElements.html (Advanced):
--------------------------------
 1a) Type non-linked objects or information in the top field and use the buttons just below that field 
 1b) Type linkable objects or information in the bottom field and use the bottons the respond by color to that field (**HINT** the bottom field's buttons don't work, inspect to get them to work. Doing this will put whomever including yourself at risk, which is why this is turned off normally.)
 2a) If you need something click "Need" but if you have something click "Have"
 2b) If you have something but need more: follow steps 2 & 3 for "Have" then repeat steps 2 & 3 for "Need"
 3a) If you Click "Need", you are done
 3b) If you Click "Have", you will need to next input how many of these you have then click "Add to Have"
 
 EXTRA) At anytime you can click on edit for holding items to adjust how many you have & almost all object names can be clicked on to rename (click change to keep all changes)
 HINT) The extra advanced input system is not turned on by default, so a user is required to alter the page to get this to work and any actions after this will be the user admitting they turned on this funciton for the feature knowing it is an at risk feature that they just turned on.
